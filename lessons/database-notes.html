Database

1. What is a Database?

A database is an organized collection of related data stored so that it can be easily accessed, managed, and updated.

Examples: School student records, bank customer accounts, hospital patient records.

Simple definition: A database is a collection of related data stored in an organized way for easy retrieval and management.

2. Key Terms in Databases

Data

Raw facts or figures. Example: 24, “James”, “Form 5”.

Information

Processed data that has meaning. Example: “James scored 86 in Mathematics.”

Record (Row)

A complete set of related fields. Example: Name, Age, Class.

Field (Column)

A single piece of data in a record.

Table

A collection of related records arranged in rows and columns.

DBMS (Database Management System)

Software used to create, store, manage, and retrieve data.

Examples: Microsoft Access, MySQL, Oracle, SQL Server.

3. Types of Databases

Flat-File Database

Stores data in a single table, like an Excel sheet.

Relational Database

Stores data in multiple related tables using keys.

Distributed Database

Data stored in different locations but appears as one system.

Object-Oriented Database

Stores data as objects used in programming.

Hierarchical Database

Data arranged in a parent-child structure.

4. Advantages of Databases

  • Reduces data redundancy
  • Increases data security
  • Easy to update and modify
  • Ensures data consistency
  • Fast data retrieval
  • Allows data sharing
  • Supports backup and recovery

5. Disadvantages of Databases

  • Can be expensive to set up
  • Requires trained personnel
  • Needs high hardware resources
  • Database failure affects many users

6. Keys in a Database

Primary Key

Uniquely identifies each record; cannot be duplicated or empty.

Foreign Key

A primary key from another table used to create relationships.

Candidate Key

A field that can qualify to be a primary key.

Composite Key

A key formed using more than one field.

7. Relationships in Databases

One-to-One

One record matches exactly one record in another table. Example: Student ↔ Locker.

One-to-Many

A single record relates to many records. Example: Teacher → Many Students.

Many-to-Many

Many records in both tables relate together. Example: Students ↔ Courses.

8. DBMS vs RDBMS

DBMS: Stores data in tables or files, no relationships.

RDBMS: Stores data in related tables using keys.

9. SQL (Structured Query Language)

DDL (Data Definition Language)

Used to create or modify structure.

Commands: CREATE, ALTER, DROP

DML (Data Manipulation Language)

Used to manipulate data.

Commands: INSERT, UPDATE, DELETE, SELECT

DCL (Data Control Language)

Controls user permissions. Commands: GRANT, REVOKE.

TCL (Transaction Control Language)

Controls transactions. Commands: COMMIT, ROLLBACK.

10. Normalization

Process of reducing redundancy and improving data integrity.

1NF

No repeating groups.

2NF

No partial dependencies.

3NF

No transitive dependencies.

11. Types of Database Users

End Users

Use the database for daily tasks.

Database Administrator (DBA)

Manages and controls the database.

Application Programmers

Develop applications that use the database.

12. Applications of Databases

  • Schools
  • Hospitals
  • Banks
  • Businesses
  • Government
  • Websites
