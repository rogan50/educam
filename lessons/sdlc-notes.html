System Development Life Cycle (SDLC)

Meaning of System Development Life Cyle

  • The System Development Life Cycle (SDLC) is the step-by-step process used to plan, design, develop, test, and maintain an information system.
  • It helps computer professionals (like system analysts, programmers, and project managers) to create reliable and efficient systems.
  • Each stage must be carefully followed to ensure the system meets the user's needs.

In short:

SDLC is the process of developing a computer system from the begining to the end.

Importance of SDLC

The SDLC is very important because it:

  • Ensures the system is well-planned and organized.
  • Helps detect and correct errors early in development.
  • Ensures the final system meets user needs and business goals.
  • Saves time and cost by following a clear structure.
  • Provides a standard guide for developers to follow.

Phases/Stages of the System Development Life Cycle

The SDLC consist of Seven main stages:

1. Problem Identification (Preliminary Investigation)

  • This is the first stage of SDLC.
  • The goal is to identify and understand the problem or need for a new system.

Activities:

  • Study the existing system.
  • Identify weaknesses, errors, or inefficiencies.
  • Define the purpose of developing a new system.
  • Prepare a brief feasibility report.

Output:

A problem statement or feasibility report recommending whether to proceed or not.

2. Feasibility Study

  • At this stage, developers analyze whether the new system is practical and possible to develop
  • It answers the question: "Can this system really be built?"

Types of Feasibility:

  1. Technical Feasibility: Do we have the right hardware and software?
  2. Economic Feasibility: Is the project affordable and cost-effective?
  3. Operational Feasibility: Will users accept and use it effectively?
  4. Legal Feasibility: Does it comply with laws and regulations?
  5. Schedule Feasibility: Can it be completed within the required time?

Output:

A feasibility report showing whether the system should go ahead.

3. System Analysis

  • This stage involves studying the current system in detail to understand how it works.
  • The goal is to gather requirements - what users want the new system to do

Activities:

  • Collect data through interviews, questionnaires, and observation.
  • Analyze workflows, inputs, processes, and outputs.
  • Identify system requirements (both functional and non-functional).
  • Prepare System Requirement Specification (SRS).

Output:

A requirement specification document describing what the new system should achieve.

4. System Design

  • This is the stage where the system's structure is created based on the requirements gathered.
  • The aim is to plan how the system will look and work.

Activities:

  • Design input forms, output reports, and user interfaces.
  • Design databases (tables, fields, and Relationship).
  • Create data flow diagrams (DFDs), flowcharts, or ER diagrams.
  • Select suitable hardware and software.

Output:

A system design document that guides programmers in the next stage.

5. System Development (Coding/Implementation)

  • This is the stage where programmers write the actual code for the system.
  • It involves converting the design into a working program using programming languages like
    • Java, Python, PHP, or C++.

Activities:

  • Write and compile program code.
  • Combine system modules (integration).
  • Document the code (add comments for future reference).

Output:

A fully functional software system (prototype) ready for testing.

6. System Testing

  • After development, the system is tested to detect and fix errors or bugs.
  • The goal is to ensure that the system works as expected.

Types of Testing

  1. Unit Testing: Tests individual modules or programs.
  2. Integration Testing: checks if all modules work together.
  3. System Testing: Tests the complete system as a whole.
  4. User Acceptance Testing (UAT): Users test the system to comfirm it meets their needs.

Ouput:

A tested system that is ready to be installed and used.

7. System Implementation

  • This stage involves installing the new system and training users to use it effectively.

Implementation Medthods:

  1. Direct Changeover: Old system is replaced by the new one immediately.
  2. Parallel Running: Both systems run together for some time untill the new one proves reliable.
  3. Phased Implementation: The system is introduced in parts or phases.
  4. Pilot Running: The system is tested in one department before full rollout.

Activities:

  • Install software and hardware.
  • Train users and provides manuals.
  • Convert old data into new system format.

Output:

A working system in real use by the organization.

System Maintenance and Review

  • After implementation, the system must be monitored, maintained, and updated to handle changes and errors.

Activities:

  • Correct errors (bug fixing).
  • Add new features when needed.
  • Update the system for new technologies or policies.
  • Conduct performance reviews.

Output:

A stable and updated system that continues to meet user needs.

Importance of SDLC to students and Organizations

  • Helps in developing high-quality systems.
  • Reduces risk of system failure.
  • Saves time and cost.
  • Encourages teamwork and documentation.
  • Ensures systems meet user needs and remain reliable.
