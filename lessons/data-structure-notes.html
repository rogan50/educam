Data structures

A data structure is a way of organizing, storing, and managing data in a computer so that it can be used effeciently.
It helps the computer process information quickly and saves memory space. In simple words, A data structure tells the computer how to store data and how to access it when needed.

Example:

If you want to store a list of student's names, you can use an array.
If you want to link one student's record to another, you can use a linked list.

Importance of Data Structures

Data structures are very important because they:

  1. The make data easy t0 access and manipulate
  2. They help to store large amounts of data efficiently.
  3. They allow faster searcing and sorting of data.
  4. They help to save computer memory
  5. The used to design efficient algorithm
  6. The are important in database design, programming, and problem solving.

Classification of Data Structures

Primitive Data Structures

These are the basic data type that come built-in with programming languages.

Examples:

  • Integer - stores whole numbers (e.g., 5, -9, 100)
  • Float/Real - stores decimal numbers (e.g., 5.25, -0.1)
  • Character - stores single letters (e.g., 'A', 'b')
  • Boolean - stores True or False
  • Pointer - stores memory address of another variable

These are the building blocks of all other complex data structures.

Non-Primitive Data Structures

These are complex data types build using primitive data types.
The are divided into two categories:

  • Linear Data Structures
  • Non-Linear Data Structures

Linear Data Structures

In a linear structure, the elements are arranged one after another - like a line.
Each elements has a next and previous element (except the first and last).

Example includes:

  1. Array
  2. Linked List
  3. Stack
  4. Queue

Array

An array is a collection of similar elements stored incontinuous memory locations.

Example

If you have marks of 5 students:
marks[5] = {75, 80, 60, 90, 88}

Each elements can be accessed using an index, starting from 0.

Advantages

  • Fast access using index (direct access)
  • Easy to sort or search

Disadvantages

  • Fixed size
  • Cannot easily insert or delete data

Use: Storing lists, tables, or matrices.

Linked List

A linked list is a collection of nodes, where each node contains:

  • Data
  • Address (link) : [Data | Next] - [Data | Next] - [Data | Next] - NULL

Advantages

  • Can easily grow or shrink in size
  • Easy to insert or delete elements

Disadvantages

  • Uses extra memory for links
  • Accessing data is slower (must go node by node)

Stack

A stack is a structure where data is arranged such that the last item added is the first to be removed.

it follows the rule:

LIFO - Last In, First Out

Example: Imagine stacking plates

  • Add plate = Push
  • Remove plate = Pop

main operations:

  • Push: Add item to top
  • Pop: Remove item from top
  • Peek: View the top item

Queue

A queue is a structure where the first item added is the first to be removed.

It follows the rule:

FIFO - First In, First Out

Example: People standing in a line - the first person to enter is the first to leave.

Main operations:

  • Ennqueue: Add to rear
  • Dqueue: Remove from front

Types of Queues:

  • Simple Queue
  • Circular Queue
  • Double-ended Queue (Deque)
  • Priority Queue

Uses:

  • Task scheduling
  • Printing jobs
  • Managing requests in operating systems

Non-Linear Data Structures

In non-linear structures, data elements are arranged sequentially.
They are connected in a hierarchical or network form.

Examples:

  1. Tree
  2. Graph

Tree

A tree is a hierarchical structure consisting of nodes.

  • The top node is called the Root.
  • The bottom nodes are called Leaves.
  • The nodes in between are Branches.

Example:

Root
        /    \
       A     B
    /    \    /    \
 C      D E     F

Graph

A graph consists of vertice (nodes) connected by edges (links)
